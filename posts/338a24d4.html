入坑神器SpaceVim之路

Vim SpaceVim
Vim玩转指北
发布日期:   2018-04-06
更新日期:   2020-12-09
文章字数:   982
阅读时长:   3 分
阅读次数:  

坚持是一个很神奇的东西，很多人都无法得到！

每一个程序员都需要一个顺手的开发工具，就我而言，我需要一个可以写 Python/Go/Shell 的开发环境。作为一个新手来说，PyCharm 绝对是一个神器，既不用折腾又好用。但是对于一个喜欢折腾的人来说，使用圣战中的 Vim 和 Emacs 绝对是装 X 神器。

  • 远看屏幕各种秀，一通键盘啪啪啪。
  • 感觉操作猛如虎，近看全都是错误。

想必对于很多人来说，能够使用 Vim 和 Emacs 作为主力的开发工具简直是不敢想象。因为太多的命令需要记忆，太多的插件需要安装和配置，太多的定制化需要自己搞一个 DotFiles 的仓库不可。但是，一旦你失败了三次还是没有放弃，并且拥有 200% 的欣喜的态度，那么进入 Vim 和 Emacs 开门就开始为你敞开了。当然对于，只会使用 hjkl 的人来说，我就不多说什么了。

为什么说 Vim 和 Emacs 谁好谁坏是一场圣战呢？喜欢的可以 Google 一下呦。那么为什么很多著名的程序员都热衷使用 Vim 和 Emacs 作为开发工具呢？就我而言的话，主要是效率和绝对自由化的定制，即为自己量身打造的似的。就好像自己准备打仗，一个是去集市买盔甲（大小很难合适），而另一个人就是自己给自己量身准备了一套盔甲，你说谁更顺手呢？当然，不言而喻了。

我在上大学的时候，折腾了很长时间的 Vim 的使用，从开始的上下左右，到后面可以灵活使用 Vim 来写 Shell 脚本。但是，开始学习 Python 的时候，越发感觉写起代码来说很吃力。没有代码提示，没有错误提示，没有项目搜索，没有很多没有。太多的没有让我开始放弃这个家伙了，投入了 PyCharm 的怀抱了。

直到大四去饿了么实习的时候，发现同事中有人用 Vim 和 Emacs 去写代码。自己感觉很酷炫，打听之后，开始使用国人编写的Python集成环境K-Vim` 工具集。说实话，但是安装就感觉很吃力，好几天被各种错误纠缠着，还请教过同事。但是，多多尝试之后就开始熟悉了，一旦错误就能够直到大致错误的原因和解决方法了。

后来因为一些原因，重新去了另一家公司，结束了这段实习的好环境，开始投入紧张的工作。随着工作量的加大，慢慢的又回到了 PyCharm 的怀抱。哈哈哈，没办法，不争气呀。

直到前段时间，董哥说他们工作中都用是 Vim 和 Emacs 去写代码，还没有见过使用 IDE 工作的人。顿时，重燃的火开始在自己心中燃烧起来了，重振旗鼓，开始倒腾。经过一番很长的时间，才弄好自己的开发环境。

  • 尘劳迥脱事非常，紧把绳头做一场。
  • 不经一番寒彻骨，怎得梅花扑鼻香。

当然，也遇到了一些坑，哎，都是眼泪。其中一个就是，我安装了 Space-Vim 的工具，但是使用在现实图标上一直无法正常显示。自己折腾了好几个晚上都没解决，感觉走投无路了。最后经过一个大神的指点，发现居然是一个个小小的配置没有到位，尴尬呀！具体修改，如下图所示。

最后，秀一下我开发环境，嘻嘻！

  • 咬定青山不放松，立根原在破岩中。
  • 千磨万击还坚劲，任尔东西南北风。
